# Daily Scratchpad - 2026-05-01

## Session Work

### Bug Fix: Stats Zeros on Post-Match Display
- **Root cause**: GEP sends `tabs: null` AFTER match end to clear post-match data. Without a guard, EventProcessor parsed null as `{}`, extracted zeros for all stats, emitted `live-stats` with zeros to renderer, which overwrote the correct match data in the match store.
- **Fix**: Added null/undefined guard at the top of the `tabs` handler in event-processor.ts. If `value === null || value === undefined`, return immediately without emitting any events.
- **Tests**: 4 new tests covering null tabs scenarios including the exact real-world sequence (correct tabs during match -> match end -> null tabs clear).

### Bug Fix: legendSelect null clear
- **Root cause**: GEP sends `legendSelect_X: null` at match end to clear. The handler would try to process null as an object and fail silently.
- **Fix**: Added null guard before the legendSelect handler that skips null/undefined values.
- **Tests**: 2 new tests for legendSelect_0/1/2 null values.

## Key Learnings
1. GEP sends null values to "clear" info updates after match end (tabs, legendSelect)
2. The tabs data is correct DURING the match - the bug was in the post-match cleanup
3. Must always guard against null/undefined values in all GEP info update handlers
4. The `live-stats` emit fires for every tabs update regardless of inMatch state, which means the renderer can get stale/zeroed data
